## Technical Decisions

### Dev Raw Layer Strategy (Sept 26)
**Decision**: Create two raw databases in Dev environment
**Rationale**: Ingestion team needs stable raw layer; engineering teams need flexibility to modify test data
**Implementation**: Zero-copy clone approach (dev_raw_clone_db) to avoid production view dependencies

### Abacus Integration Architecture (Oct 1)
**Decision**: Ingest Abacus Bronze/Silver data into our Raw layer vs. consuming directly from their Silver layer
**Rationale**:
- Maintain Data Vault 2.0 methodology consistency
- Retain control over integration logic and security (tenant ID, etc.)
- Enable AI-assisted code generation for business rules refactoring (on-prem â†’ dbt)
**Impact**: Slightly more work upfront, but positions us for automated code generation workflow

### Repository Split Strategy (Sept 26)
**Decision**: Split EDP_Data_Domains into 3 repos by release cadence
**Rationale**: Different teams have different deployment schedules (streaming vs. batch vs. analytics)
**Implementation**: Leverage dbt mesh and cross-project references
